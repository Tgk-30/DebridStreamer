# YAWF Stream v0.9.19 Verification Log

Date: 2026-07-19

Release tag: `v0.9.19-web`

Website: `https://tgk30.com/debridstreamer/`

This log records the behavior and release gates verified for v0.9.19. It does not contain credentials, private prompts, private provider responses, or private service URLs.

## Player reliability and performance

- Verified fatal HLS network failures retry twice with bounded backoff before showing recovery controls.
- Verified fatal HLS media failures receive one media recovery attempt before showing recovery controls.
- Verified Retry playback creates a fresh player attempt without closing the player.
- Verified unsupported HLS produces an actionable recovery panel in browser and desktop modes.
- Verified the desktop recovery panel is not replaced by the normal bundled-player launch state.
- Verified native player, external player, fallback, recovery, mount, and unmount events are recorded as bounded diagnostics.
- Verified player resize work is limited to one animation-frame update at a time.
- Verified pointer activity coalesces chrome timeout resets instead of creating a new timer for every pointer event.
- Verified the player and player information dialog trap focus, close with Escape, and restore focus.
- Verified the up-next countdown no longer causes repeated live-region announcements.

## Provider smoke tests and cache behavior

- Added read-only account and cache smoke checks for configured Real-Debrid, AllDebrid, Premiumize, and TorBox services.
- Verified each in-app check has a 10 second timeout and one provider failure does not block the remaining providers.
- Verified the checks do not add magnets, select files, or delete provider data.
- Added a standalone smoke runner that reads provider tokens only from environment variables.
- Verified provider tokens are sent in request headers and never placed in URLs or result output.
- Verified response bodies are not included in failure reports.
- Added a manual `provider-smoke` GitHub Actions workflow for credentialed checks.
- Added provider smoke contract tests to CI.
- Corrected Premiumize cache checks in the web and Swift services to use the documented POST form request.
- Verified Premiumize cache hashes and credentials are sent in the form body and not the query string.
- Credentialed live provider checks were not run locally. They remain an explicit manual release check because no provider credentials are stored in the repository.

## Diagnostics

- Added a bounded in-memory diagnostic buffer limited to 100 recent events.
- Added structured app, player, provider, and storage diagnostic categories.
- Verified bearer tokens, key assignments, credential-like query values, and long credential-shaped values are redacted.
- Added a downloadable JSON support report with app version, runtime, platform, safe configuration state, provider types, source types, and recent redacted events.
- Verified support reports do not include provider token values, API key values, configured server URLs, or private provider responses.
- Added Settings troubleshooting controls for provider smoke checks and diagnostic export.
- Verified the browser clearly explains that direct provider checks require the desktop app.
- Verified the troubleshooting area uses ordinary text status and no decorative live indicators.

## Accessibility

- Added pressed-state semantics to search, browse, watchlist, debrid-library, caption-color, and related selection controls.
- Added expanded-state and dialog relationships to browse and player controls.
- Added accessible labels to caption search, setup reminders, and featured-title navigation.
- Added alert semantics to caption and external playback failures.
- Verified the player recovery action is keyboard reachable and has a clear accessible name.
- Verified the Settings troubleshooting controls remain readable and keyboard reachable.

## Cleanup and release consistency

- Removed stale experimental-player comments and updated the embedded-player graduation documentation.
- Marked old feature-gap documents as historical snapshots so they are not mistaken for current release state.
- Updated the web app, server, Tauri app, Rust package, website package, and displayed website build to v0.9.19.
- Verified user-facing copy added in this release contains no em dash characters.
- Verified the Settings Updates view, troubleshooting section, Reset and uninstall section, and footer at a desktop viewport.
