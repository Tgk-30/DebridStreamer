# YAWF Stream v0.9.20 Verification Log

Date: 2026-07-19

Release tag: `v0.9.20-web`

Website: `https://tgk30.com/debridstreamer/`

This log records the migration, installation, and security work verified for
v0.9.20. It does not contain credentials, private service responses, session
values, or raw provider URLs.

## Clean installation

- Added a reusable `clean-install` GitHub Actions workflow that downloads the
  completed release assets onto fresh runners.
- Added native Apple Silicon and Intel Mac DMG installation checks.
- The macOS checks copy the app from the mounted DMG, verify Developer ID and
  Gatekeeper acceptance, validate stapling, reject Homebrew-linked binaries,
  boot the bundled server, and launch with an empty home directory.
- Added Windows MSI silent installation, installed-resource discovery, bundled
  libmpv validation, server boot, and clean-profile launch.
- Added Linux AppImage extraction, bundled libmpv validation, server boot, and
  clean-profile launch before any deb package is installed.
- Added Linux deb installation through apt, installed resource discovery,
  server boot, and clean-profile launch.
- Connected clean installation to the tagged release workflow after all draft
  assets are present. The draft must not be published unless these jobs pass.

## Migration safety

- Replaced the repeated server migration branches with an ordered append-only
  migration registry.
- Each server migration and its version marker now share one immediate SQLite
  transaction.
- A failed migration rolls back every schema change and leaves the prior
  version marker intact.
- The server refuses databases created by a newer unsupported app.
- Added immutable SHA-256 snapshots for every released server migration.
- Added v1 and v7 server fixtures that preserve accounts, profiles, sessions,
  settings, encrypted credential rows, watchlist, history, folders, library,
  streams, requests, and audit records through the v8 schema.
- Added v1 and v5 IndexedDB fixtures that preserve settings, desktop secrets,
  watchlist, history, folders, library, taste events, and downloads through the
  current v6 schema.
- Verified the settings localStorage and desktop keychain migration suites with
  the new fixtures.

## Security decisions

- Added `docs/SECURITY_DECISIONS.md` with ten explicit decisions, their trust
  boundaries, enforcement, and accepted beta risks.
- Added an executable security gate covering the decision log, desktop CSP,
  Tauri permissions, updater trust, production raw-stream defaults, AES-256-GCM
  use, generated key permissions, CSRF, transactional migrations, and
  diagnostics redaction.
- Removed the default Tauri opener permission, which also exposed file reveal.
  External opening is now scoped to HTTP, HTTPS, and the Windows installed-app
  settings page.
- Replaced the default Tauri process permission with restart only, which is the
  single command needed after an updater install.
- Added a server test that requires Secure and SameSite attributes on both auth
  cookies while keeping only the CSRF cookie readable to the client.
- Recorded unsigned Windows installers as an accepted v0.9 beta risk and a hard
  v1 blocker. macOS signing and notarization remain mandatory.
