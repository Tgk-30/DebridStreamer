Self-host

Run one private server for every screen.

Use an always-on desktop, NAS, VPS, Raspberry Pi, or private home server. Server Mode owns profile isolation, encrypted credentials, stream sessions, bandwidth visibility, and the provider-facing IP.

The desktop app can start Server Mode from Settings, then expose a hosted PWA for phones, tablets, and other computers with a copyable setup URL and QR code.

Hosting model

Lightweight by default, heavier only when you choose it.

The public site can stay mostly static. Your private server handles account setup, cached metadata, and provider handoffs while users bring their own debrid keys.

Static publishing

Docs, release pages, and platform downloads can sit behind ordinary static hosting or GitHub Pages.

Cached metadata

TMDB is hit for searches, cache misses, and stale refreshes; normal browsing reuses local metadata.

BYOK provider traffic

Debrid and subtitle provider access stays tied to each user's own credentials instead of a shared hosted bill.

Updates

Signed OTA updates for the pre-bundled app.

The desktop app checks GitHub Releases for a signed latest.json. When a new version is available, the in-app updater can download, install, and relaunch into the new build. Auto-checks can be controlled from Settings.
